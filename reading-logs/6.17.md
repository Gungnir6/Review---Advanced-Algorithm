# 阅读笔记

今天继续看 Section III，重点放在 rotated radix trie 的负载分析上

昨天只是大概知道它会对 trie 的节点数组做随机旋转，今天进一步理解了论文为什么要引入 ball 和 bin 的说法。trie 中每个非空指针可以看成一个 ball，所有旋转后叠加到一起的数组位置可以看成 bin

如果一个 ball 来自源节点 $s$，原来的 child index 是 $c$，那么它会被放到

$$
\phi(s,c) = (c + r_s) \bmod n
$$

这里 $r_s$ 是节点 $s$ 的随机旋转量

今天比较重要的理解是 load 的含义：一个 bin 的 load 就是有多少个 ball 被映射到了这个位置。论文要证明的是每个 bin 的 load 不会太大，最好能控制在 $polylog(n)$ 以内

如果 load 能被控制住，那么每个 bin 就可以用 dynamic fusion node 来存这些小规模内容，这样查询和插入仍然可以保持常数时间

目前对 Chernoff bound 的具体推导还没有完全看懂，但大体思路清楚了：对固定的 bin 来说，期望 load 是 $O(1)$，然后用概率界说明它超过 $polylog(n)$ 的概率很小
