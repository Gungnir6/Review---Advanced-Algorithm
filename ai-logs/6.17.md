# 1. rotated radix trie 中的 ball 和 bin 分别是什么

在 Section III 中，论文为了分析 rotated radix trie，把 trie 中的非空指针抽象成 **ball**，把合并后的数组位置抽象成 **bin**。

具体来说，普通的 $n$-ary radix trie 中，每个内部节点都有一个长度为 $n$ 的数组。如果某个位置存放了指向孩子节点或叶子的指针，那么这个非空位置就可以看成一个 ball。

每个 ball 可以用一个二元组表示：

$$
(s,c)
$$

其中：

- $s$ 表示这个 ball 来自第几个内部节点；
- $c$ 表示它在该节点数组中的原始 child index。

rotated radix trie 会把所有内部节点的数组旋转后叠加到一个总数组 $A$ 上。这个总数组中的每个位置就是一个 bin。

# 2. rotated radix trie 如何把 ball 映射到 bin

每个内部节点 $s$ 会有一个随机旋转量 $r_s$。如果某个 ball 在原始数组中的 child index 是 $c$，那么旋转之后它会被放到：

$$
\phi(s,c) = (c + r_s) \bmod n
$$

这个位置。

这里的关键是：同一个源节点中的所有 ball 会被整体旋转，而不是每个 key 单独哈希。不同源节点有不同的随机旋转量，因此它们叠加到同一个数组时，冲突会被随机打散。

# 3. load 是怎么定义的

论文中关心的是每个 bin 里最终会落入多少个 ball。对于某个 bin $j$，可以定义随机变量：

$$
Y_j = \sum_{i=1}^{m} X_{i,j}
$$

其中 $X_{i,j}$ 表示第 $i$ 个内部节点是否有 ball 被旋转到 bin $j$。

所以 $Y_j$ 就是 bin $j$ 的 load，也就是这个位置需要存储多少个来自不同节点的指针。

如果某个 bin 的 load 太大，那么这个位置就不能再用小规模字典结构常数时间处理；如果所有 bin 的 load 都能控制在 $polylog(n)$，就可以用 dynamic fusion node 来存储每个 bin 里的内容。

# 4. 为什么最大 load 可以被控制住

对固定的 bin $j$ 来说，每个 ball 落到这个位置的概率大约是 $1/n$。由于 trie 中总共有 $O(n)$ 个 ball，所以期望 load 是：

$$
E[Y_j] = O(1)
$$

虽然同一个节点中的不同 ball 不是完全独立的，但不同源节点的随机旋转量是独立的。因此对固定 bin 的分析仍然可以把 $Y_j$ 看成一组独立指示变量的和。

这样就可以使用 Chernoff bound 来说明：某个 bin 的 load 超过 $polylog(n)$ 的概率很小。论文进一步说明，这能得到略微次多项式级别的失败概率。

# 5. dynamic fusion node 在这里起什么作用

rotated radix trie 把多个节点数组叠加成一个长度为 $n$ 的数组后，同一个 bin 可能要存多个 ball。

论文的做法是：每个 bin 不只是一个普通槽位，而是用一个 **dynamic fusion node** 来实现。dynamic fusion node 可以在常数时间内处理规模为 $polylog(n)$ 的小集合。

因此只要能证明每个 bin 的 load 不超过 $polylog(n)$，那么每个 bin 内部的查询、插入等操作就仍然可以看成常数时间。

这也是为什么负载分析很重要：它直接决定 rotated radix trie 能不能支持高概率常数时间操作。

# 6. Section III 得到了什么结论

Section III 最后得到的结论是：rotated radix trie 是一个随机化的线性空间字典，可以存储最多 $n$ 个 $\Theta(\log n)$ 位的 key/value pair，并且每个操作以高概率在常数时间内完成。

不过它本身还不是论文最终想要的最强结果。它只能得到略微次多项式级别的失败概率，还没有达到后面 amplified rotated trie 的

$$
\frac{1}{n^{n^{1-\varepsilon}}}
$$

也没有做到 budget rotated trie 那种只使用很少随机比特的效果。

因此 Section III 更像是基础工具：先证明随机旋转数组这一思路可行，然后 Section IV 和 Section V 再分别在失败概率和随机比特数量上继续优化。
