# 2. 三个 rotated trie 结构之间是什么关系

这三个结构可以看成同一条技术路线上的不同层次。

**rotated radix trie** 是基础结构，说明可以不用传统哈希函数，而是通过随机旋转 trie 节点数组来实现类似哈希表的效果。

**amplified rotated trie** 在此基础上追求更低失败概率。它通过降低 fanout、处理 overflow ball，并用 McDiarmid 不等式分析溢出数量，把失败概率降到非常小。

**budget rotated trie** 则换了一个目标：不追求极低失败概率，而是希望减少随机比特数量。它通过减少内部节点、改变 ball-to-bin 映射，以及使用 gradually-increasing-independence hash functions 来降低随机性消耗。

# 3. succinct dictionary 为什么重要

哈希表除了时间复杂度和失败概率外，空间效率也很重要。

如果一个字典存储来自全集 $U$ 的 $n$ 个键，那么信息论下界大致是：

$$
B(|U|,n)=\log { |U| \choose n }
$$

succinct dictionary 的目标是把空间压到接近这个下界，也就是只使用

$$
(1+o(1))B(|U|,n)
$$

这样的空间。

论文中提到，rotated trie 相关结构可以通过黑盒转换得到 succinct dictionary，并且概率保证基本保留。这说明论文的贡献不只是失败概率或随机比特数量，还包括空间效率。

# 4. 为什么 budget rotated trie 和 succinctness 有联系

论文中提到，succinct dictionary 的转换会用到 budget rotated trie 作为关键组件。

直观理解是，budget rotated trie 本身随机比特使用少，并且保持线性空间和高概率常数时间操作，因此更适合作为空间压缩转换中的子结构。

这也说明 Section V 不是一个孤立结果。它一方面研究随机比特预算，另一方面也为后面的 succinct 化提供工具。
